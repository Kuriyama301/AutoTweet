# AutoTweet

AI活用型Xエンゲージメント自動化ツール

---

## 概要

**AutoTweet**は、Grok AIを活用してXでのエンゲージメントを効率化するツールです。リプライ対象のポストを自動で選定し、AI生成の文面案を提示。人間の最終判断を挟むことで、品質を保ちながら効率的にインプレッションを向上させます。

## 目的

### ビジネスゴール
- Xでアクティブな仲間（日頃から「いいね」や「リプ」をくれる人）を増やす
- 普段のツイートのインプレッションを向上させる
- 自社サービスへの流入を増やす

### 解決する課題
- 手動でのリプライ作業が負担
- 日頃からリプを頑張るのは辛い
- そもそもあまりXを開かない
- 他人に興味が持てない
- でも売上を増やすチャネルとしてインプレッションを持っておきたい

## 主な機能

### POC版
1. **ポスト検索・選定**
   - X検索URLから対象ポストをピックアップ
   - 例: 「受託開発&最新」の検索結果
   - 経営者のポストと思われるものを10件選定

2. **AIリプライ生成**
   - Grok AIを使ってリプライ文面を自動生成
   - ユーザーのアカウントになりきった文面

3. **ユーザー判断**
   - ブラウザ上で提案を確認
   - アクション選択:
     - 提案を受け入れて実行
     - 文面を修正して実行
     - スキップ

4. **リプライ実行**
   - 決定した文面でリプライ投稿
   - いいねも同時実行

### POC要件
- **実行方法**: スクレイピング or X API（検討中）
- **ユーザー判断**: ブラウザで判断可能
- **提案数**: 1日20件

### 将来の拡張（本番化時）
- Slack API連携（Slack上で通知 + ユーザー判断）
- バッチ処理（定期実行）
- 複数検索クエリ対応
- リプライ履歴管理・分析

## 技術スタック

### フロントエンド
- Next.js 14 (App Router)
- React 18 + TypeScript
- Tailwind CSS

### バックエンド
- Node.js + Express
- TypeScript
- RESTful API

### AI
- Grok API（リプライ文面生成、ポスト選定支援）

### X連携
- **POC**: スクレイピング（Puppeteer or Playwright）
  - 個人利用・補助ツールとしての位置づけ
  - 人間の最終判断を挟むため完全自動ボットではない
- **将来**: 必要に応じてX API（Basic $100/月）への移行を検討

### 開発環境
- Docker Compose

### データベース
- **POC**: 不要（またはSQLite）
- **将来**: リプライ履歴管理が必要になったら追加（MongoDB or PostgreSQL）

### テスト
- Jest（ユニットテスト）
- React Testing Library（コンポーネントテスト）
